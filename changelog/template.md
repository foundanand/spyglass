# Changelog Template

All notable changes to this project are documented in this folder, one file per
change-set. This project adheres to [Semantic Versioning](https://semver.org/)
and takes cues from [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## Naming Convention

- **Format**: `NNN-feature-name.md` (e.g. `001-replay-inspector.md`, `002-auth-refactor.md`)
- **Numbering**: Sequential, zero-padded (001, 002, 003, …)
- **Name**: Descriptive kebab-case name of the feature/change

Skip whichever of the sections below don't apply — don't leave empty headings.

**How to use this template:**

1. Copy this file to `NNN-feature-name.md` (replace NNN with the next sequential number).
2. Group changes by type for clarity; delete unused sections.
3. Document breaking changes and deprecations clearly.
4. Keep entries human-readable and actionable — describe behaviour, not just diffs.
5. Include a summary section explaining the overall purpose and the files touched.
6. Write the entry before the change is pushed.
